### 배포
- 배포는 개발자에게 가장 쉬운 기능이어야 한다.
- 배포는 무조건 자동화해야 한다.
  - 배포 명령의 자동 실행
  - 단계별 성공 여부 보고와 재실행 시 같은 결과 보장
- 배포 전략
  - 롤링 배포
    - N대씩 배포 진행
    - 롤백이 어려움
  - 블루/그린 배포
    - 다운타임을 최소화 하는 방식
    - 기존 버전 서버들을 새 버전 서버들로 교체
  - 카나리 배포
    - 일부 서버에 새 버전을 배포하고 문제가 발생하는지 확인
    - 문제가 없을 시 롤링/블루그린 배포 진행
- 무정지 배포의 의미
  - 일말의 어떤 오류도 발생하지 않는 방식이 아니라, 오류가 아주 짧게만 발생하는 방식이라는 의미
  - 짧은 시간 안에 서비스를 정상으로 제공하면 장애로 인식하지 않는다.
  - 롤링 배포 시 배포중인 서버를 로드밸러서에서 제외해서 장애의 아주 작은 가능성까지 최소화 할 수 있다.
 
### 테스트
- QA가 없는 조직에서 최소한 반드시 진행해야 할 테스트
  - 단위테스트
    - 작은 단위의 메서드, 함수, 클래스에 대한 검증
  - E2E 테스트
    - 사용자 관점에서 서비스가 제대로 동작하는지 확인
    - 모든 구성 컴포넌트가 통합적으로 제대로 동작하는지

### 캐시
- 성능향상의 효과적이고 쉬운 방법
- json 형태 추천
- 캐시 전략
  - 캐시어사이드 패턴
  - 리드스루 패턴
  - 라이트백 패턴
- 캐시 크래시, 캐시 스탬피드 주의
- 서버를 여러대 두어 SPOF 제거

### 데이터의 분산 저장
- 범위 샤딩: 순서대로 데이터를 저장
- 모듈러 샤딩: 홀짝 방식으로 데이터를 분배
- 안정해시: 분산 환경에서 데이터를 효율적으로 배치하는 방식 중 하나

### 비동기
- 배압
  - 큐를 활용해 요청 수 제한
  - 요청 제한을 통해 요청 수 제한

### 로그 수집
- 기본적으로 타임스탬프, 로그레벨, 메시지, 이벤트로 구성
- 역할
  - 오류와 문제 해결
  - 성능분석과 최적화
  - 보안, 이상 탐지
 
### 모니터링
- cpu, 메모리, 디스크, 요청 수, 에러 수, 응답 시간

### 통계
- BI: 서비스 성장 및 고객의 특성등의 서비스 통계정보 추출, 분석, 시각화 작업

### 설정 변경 가능한 설계
- 가능한 재배포 없이 서비스 동작을 제어할 수 있는 설계 권장

### 안전한 프로토콜의 설계
- 로미오와 줄리엣 버그: 서버 타임아웃 > 클라이언트 타입아웃으로 설정
